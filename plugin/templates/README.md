# Navigation

**Edit this file** when you adapt the project shape to your own needs. It's a reference for you and anyone new to the project.

## Core files

- **CLAUDE.md** — working memory. Claude reads this at the start of every conversation here.
- **overview.md** — narrative pitch. Keep it under two pages; update when the concept shifts.
- **mvp.md** — current spec. Scope, phases, open gates, what we're not doing yet.
- **log.md** — append-only timeline. Brain dumps go here with date headings.
- **open-questions.md** — running list grouped by domain (platform, product, business, privacy, meta).

## Folders

- **concepts/** — one numbered markdown file per major component (deep-dives, frameworks, systems).
- **ideas/** — dated raw brain dumps, verbatim, nothing structured yet.
- **technical/** — platform facts, API notes, glossaries, architecture decisions.

## First steps

1. Fill in the "What this project is" section in CLAUDE.md.
2. Draft a one-paragraph overview in overview.md.
3. Sketch your near-term phases in mvp.md.
4. Start a conversation with Claude — it will load CLAUDE.md automatically.

## Workflow

Brain dump → log.md → ideas/ → concepts/. Distil in-session. Don't delete, supersede. See CLAUDE.md for the full workflow.
